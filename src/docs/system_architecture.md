# 大学答辩管理系统设计文档

## 一、系统概述

### 1. 系统名称
大学答辩管理系统（University Defense Management System，UDMS）

### 2. 系统定位
该系统是一个专门用于管理大学毕业论文答辩过程的信息化系统，主要用于处理答辩计划、答辩分组、答辩评分、答辩记录和成绩统计等功能。

### 3. 系统目标
- 实现答辩过程的信息化管理，提高工作效率
- 规范答辩流程，确保答辩的公平、公正、公开
- 提供便捷的答辩信息查询和统计功能
- 提升用户体验，减少人工操作

## 二、技术架构

### 1. 技术栈选择

#### 前端技术
- Vue 3：渐进式JavaScript框架，提供响应式数据绑定和组件化开发
- Element Plus：基于Vue 3的UI组件库，提供丰富的UI组件
- TypeScript：静态类型检查的JavaScript超集，提高代码质量和可维护性
- Pinia：Vue 3的状态管理库，用于管理应用的全局状态
- Vue Router：Vue 3的路由管理库，用于实现单页应用的路由功能

#### 后端技术
- Node.js：基于Chrome V8引擎的JavaScript运行环境
- Express：轻量级的Web应用框架，用于构建RESTful API
- TypeScript：静态类型检查的JavaScript超集，提高代码质量和可维护性
- MySQL：关系型数据库，用于存储系统数据
- Redis：高性能的键值存储数据库，用于缓存和会话管理
- JWT：JSON Web Token，用于实现无状态的身份认证

### 2. 系统分层结构

#### 前端架构
- **表现层（Presentation Layer）**：负责用户界面的展示和用户交互
- **业务逻辑层（Business Logic Layer）**：负责处理前端的业务逻辑
- **数据访问层（Data Access Layer）**：负责与后端API进行通信

#### 后端架构
- **API层（API Layer）**：负责处理HTTP请求和响应
- **业务逻辑层（Business Logic Layer）**：负责处理系统的业务逻辑
- **数据访问层（Data Access Layer）**：负责与数据库进行交互
- **工具层（Utility Layer）**：负责提供通用的工具和服务

### 3. 系统模块划分

#### 核心功能模块
1. **用户管理模块**：负责用户的认证和授权
2. **学生管理模块**：负责学生信息的管理
3. **教师管理模块**：负责教师信息的管理
4. **答辩计划模块**：负责答辩计划的制定和管理
5. **答辩分组模块**：负责答辩小组的组建和管理
6. **答辩评分模块**：负责答辩成绩的录入和管理
7. **答辩记录模块**：负责答辩过程的记录和管理
8. **成绩统计模块**：负责答辩成绩的统计和分析
9. **通知公告模块**：负责系统通知的发布和管理

#### 支撑功能模块
1. **日志管理模块**：负责系统日志的记录和管理
2. **权限管理模块**：负责系统权限的管理和控制
3. **数据备份模块**：负责系统数据的备份和恢复
4. **系统监控模块**：负责系统运行状态的监控和管理

## 四、系统数据模型设计

- password：密码
- nickname：昵称

- phone：手机号
- email：邮箱
- user_group：用户组（学生/教师/管理员）
- user_admin：是否是管理员
- state：用户状态
- create_time：创建时间
- update_time：更新时间

### 2. 学生表（student）

- student_gender：性别

- phone：手机号
- email：邮箱
- create_time：创建时间
- update_time：更新时间

### 3. 教师表（teacher）
- teacher_id：教师ID，主键

- update_time：更新时间

- update_time：更新时间

- group_id：小组ID，主键

- update_time：更新时间

### 7. 小组教师表（group_teacher）
- gt_id：ID，主键

- update_time：更新时间

### 8. 小组学生表（group_student）
- gs_id：ID，主键

- thesis_title：论文题目
- order_num：答辩顺序
- status：状态（待答辩/已答辩）

- update_time：更新时间

- update_time：更新时间

### 11. 答辩记录表（defense_record）
- record_id：记录ID，主键
- gs_id：小组学生ID，外键
- record_content：记录内容
- record_files：记录文件
- create_time：创建时间
- update_time：更新时间

### 12. 通知公告表（notice）
- notice_id：通知ID，主键
- notice_title：通知标题
- notice_content：通知内容
- notice_type：通知类型
- publish_time：发布时间
- publisher：发布人ID，外键

- update_time：更新时间

## 六、系统部署方案

### 1. 服务器配置
- 前端服务器：Nginx
- 后端服务器：Node.js + Express
- 数据库服务器：MySQL
- 缓存服务器：Redis

### 2. 部署架构
- 前端服务器：部署静态资源，处理HTTP请求
- 后端服务器：部署API服务，处理业务逻辑
- 数据库服务器：存储系统数据
- 缓存服务器：缓存热点数据，提高系统性能

### 3. 安全措施
- 数据加密：对敏感数据进行加密存储
- 访问控制：基于角色的访问控制（RBAC）
- 防止攻击：防止SQL注入、XSS攻击等
- 备份恢复：定期备份数据，确保数据安全
